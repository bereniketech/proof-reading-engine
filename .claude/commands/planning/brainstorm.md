# Brainstorm

Invoke the brainstorming skill to collaboratively explore an idea, ask clarifying questions, propose approaches, and produce an approved design spec before any implementation begins.

## Instructions

1. Load and follow the brainstorming skill from `skills/_studio/brainstorming/SKILL.md`
2. Start by exploring the current project context (files, docs, recent commits)
3. Ask one clarifying question at a time to understand purpose, constraints, and success criteria
4. Propose 2-3 approaches with trade-offs and your recommendation
5. Present the design in sections, getting approval after each
6. Write the approved design to `docs/specs/YYYY-MM-DD-<topic>-design.md`
7. Run spec self-review (placeholders, consistency, scope, ambiguity)
8. Ask the user to review the written spec
9. Once approved, transition to planning

**Rule:** Do not write any implementation code until the design is approved.
